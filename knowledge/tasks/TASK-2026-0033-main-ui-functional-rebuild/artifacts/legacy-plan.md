# Возврат рабочего web UI на основной маршрут

- Дата: 2026-04-02
- Статус: done
- Источник: замечания пользователя по текущему основному GUI `/`, где вместо рабочего интерфейса отдавался перенесённый концепт

## Цель

Убрать с основного маршрута презентационный Clash-ориентированный экран и вернуть пользователю рабочую панель управления, где доступны реальные действия, импорт, подписки, runtime, профили и узлы.

## Изменения

- Вернуть на `/` и `/index.html` встроенный рабочий UI `INDEX_HTML`, который уже поддерживает реальные действия и все основные сценарии управления.
- Оставить `gui/design_review.html` отдельным review-маршрутом `/design-review`, чтобы концепт не блокировал основную работу.
- Сохранить legacy-алиасы как fallback-маршруты, не ломая существующий доступ к старому встроенному экрану.
- Обновить тесты так, чтобы они проверяли разделение между root-route и review-route, а также наличие реальных контролов в основном UI.

## Проверки

- `python3 -m py_compile gui/gui_server.py`
- `python3 -m unittest tests.test_gui_server`
- Локальный HTTP smoke:
  - открыть `/`;
  - проверить, что на главном маршруте доступны рабочие кнопки `Старт`, `Стоп`, `Снять диагностику`;
  - проверить, что импорт, подписки, runtime, профили и узлы доступны прямо на `/`;
  - открыть `/design-review` и убедиться, что концепт остаётся отдельным review-экраном.

## Допущения

- Наиболее безопасный способ быстро вернуть рабочий сценарий — перестать отдавать концепт на main-route, а не пытаться срочно доводить его до production-качества в той же итерации.
- Текущий backend API и встроенный `INDEX_HTML` уже покрывают нужный рабочий сценарий без изменений HTTP-контуров.
- Review-экран остаётся полезным как отдельный маршрут для дальнейшей UI-итерации, но не должен быть основной точкой входа.

## Итог

- На `/` и `/index.html` возвращён встроенный рабочий интерфейс с реальными действиями `Старт`, `Стоп`, `Снять диагностику`, а также с доступом к импорту, подпискам, runtime, профилям и узлам.
- `gui/design_review.html` сохранён как отдельный маршрут `/design-review`, чтобы концепт оставался доступным для дальнейшей доработки, но не мешал основной работе.
- Тесты обновлены под новую схему маршрутов и проверяют:
  - разделение между root-route и review-route;
  - наличие реальных контролов в основном UI;
  - сохранение review-asset как отдельного экрана.
- Подтверждено:
  - `python3 -m py_compile gui/gui_server.py`;
  - `python3 -m unittest tests.test_gui_server`;
  - локальный HTTP smoke на временном сервере `127.0.0.1:8422` с проверкой ответов для `/` и `/design-review`.
