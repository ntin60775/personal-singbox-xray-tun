## Цель

Убрать жёсткую absolute-path привязку из `Icon=` в desktop launcher и сделать так, чтобы launcher брал иконку из локального `assets/subvost-xray-tun-icon.svg` текущего bundle через поддерживаемый user-local icon lookup.

## Итог

`subvost-xray-tun.desktop` и installer для пользовательского меню переведены на `Icon=subvost-xray-tun-icon` без absolute path в самом launcher. В `lib/subvost-common.sh` helper `subvost_sync_desktop_launcher_icon` теперь синхронизирует user-local иконку в `XDG_DATA_HOME/icons/hicolor/scalable/apps/` как ссылку на текущий `assets/subvost-xray-tun-icon.svg`, а затем нормализует `Icon=` в tracked launcher и в установленной пользовательской копии, если она существует и доступна на запись.

Локально подтверждены `bash -n` для root-level и внутренних shell-скриптов, `python3 -m py_compile` для backend-модулей, точечный `unittest` на desktop launcher и `markdown-localization-guard` для затронутых Markdown-файлов. За счёт этого launcher остаётся без жёсткой path-привязки, но иконка по-прежнему приходит из локального bundle-ассета. Остаточный риск один: без ручной проверки в конкретной desktop-среде нельзя на 100% подтвердить момент обновления визуального кеша иконок, хотя контракт `.desktop` и XDG-lookup теперь соответствует поддерживаемой схеме.
