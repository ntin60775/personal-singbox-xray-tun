## Цель

Зафиксировать отдельным подконтуром визуальный контракт нативного `GTK4` клиента на основе `Raycast DESIGN.md`, чтобы следующие этапы `TASK-2026-0053` строились в одной тёмной desktop-shell системе, а не распадались на несогласованные стили экранов `Dashboard / Subscriptions / Log / tray / settings`.

## Подсказка по статусу

Использовать только одно из значений:

- `черновик`
- `готова к работе`
- `в работе`
- `на проверке`
- `ждёт пользователя`
- `заблокирована`
- `завершена`
- `отменена`

## Границы

### Входит

- выбор и фиксация `Raycast` как основного dark reference;
- визуальный контракт для:
  - `Dashboard`
  - `Subscriptions`
  - `Log`
  - системного трея
  - окна настроек
- палитра, уровни поверхностей, акценты, типографика, ритм отступов и состояния компонентов;
- критерии, которые отличают desktop-shell от обычного web-dashboard.

### Не входит

- реализация новых `GTK4` виджетов и экранов;
- подключение service-layer, runtime и routing-логики;
- замена launcher-а или переключение проекта на native UI как основной GUI;
- рефакторинг существующего кода `gui/native_shell_app.py` в этой подзадаче.
