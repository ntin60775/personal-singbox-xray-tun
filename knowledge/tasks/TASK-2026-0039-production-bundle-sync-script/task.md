## Границы

### Входит

- новый публичный wrapper и внутренняя shell-реализация для установки/обновления bundle-каталога;
- безопасная синхронизация рабочего дерева в внешний production-каталог;
- обновление README и knowledge-артефактов под новый deploy-сценарий.

### Не входит

- system-wide installer для `xray` или systemd поверх уже существующего `install-on-new-pc.sh`;
- автоматическое удаление произвольных stale-файлов из production-каталога без отдельного подтверждения;
- ручной сетевой smoke уже обновлённого production bundle.

## Итог

В корень bundle добавлен публичный wrapper `install-or-update-bundle-dir.sh`, а в `libexec/` — его реализация. Скрипт принимает абсолютный target path аргументом или через `SUBVOST_BUNDLE_TARGET_DIR`, копирует bundle через `tar`-sync, не переносит `.git`, `.codex`, `.playwright-cli`, `.gitignore`, `AGENTS.md` и кэши, сохраняет содержимое `logs/` в целевом каталоге и после копирования актуализирует `Icon=` в `subvost-xray-tun.desktop` под новый путь bundle.

README обновлён: теперь в нём описан отдельный сценарий установки или обновления bundle в внешний каталог без ручного копирования файлов. Локально пройдены `bash -n *.sh`, `bash -n libexec/*.sh`, `bash -n lib/*.sh`, `python3 -m py_compile gui/gui_server.py gui/subvost_runtime.py gui/subvost_store.py gui/subvost_parser.py` и `python3 -m unittest tests.test_gui_server`. Скрипт дополнительно проверен на временном каталоге `/tmp`, после чего им синхронизирован production-путь `/opt/subvost-xray-tun/`.

Остаточный риск: deploy-сценарий по умолчанию не удаляет stale-файлы, которых больше нет в исходном дереве. Это сделано намеренно ради безопасного обновления production-каталога без неявных удалений.
