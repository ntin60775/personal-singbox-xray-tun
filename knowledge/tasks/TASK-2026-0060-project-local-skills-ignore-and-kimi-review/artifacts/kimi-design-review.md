# Консультационное дизайн-ревью Kimi CLI

## Контекст запроса

Kimi CLI получил на ревью макет вкладки `Маршруты` для desktop VPN/TUN приложения:

- верхние вкладки: `Подключение`, `Подписки`, `Маршруты`, `Диагностика`, `Настройки`;
- новый экран показывает отчёт по правилам обхода VPN;
- слои отчёта: встроенные template-правила, активный профиль, итоговый runtime, конфликты;
- приоритет правил: `template > профиль > catch-all`;
- проверяемый заголовок блока: `Исключения прямого обхода`.

Команда:

```bash
kimi --work-dir /home/prog7/РабочееПространство/projects/PetProjects/personal-singbox-xray-tun --skills-dir /home/prog7/РабочееПространство/projects/PetProjects/personal-singbox-xray-tun/.agents/skills/ui-ux-pro-max --skills-dir /home/prog7/РабочееПространство/projects/PetProjects/personal-singbox-xray-tun/.agents/skills/playwright-interactive --quiet --plan -p 'Только текстовое дизайн-ревью, без изменения файлов и без запуска дополнительных команд. Контекст: desktop VPN/TUN app на русском, верхние вкладки: Подключение, Подписки, Маршруты, Диагностика, Настройки. Новая вкладка Маршруты должна показывать отчет по правилам прямого обхода VPN. Заголовок блока: Исключения прямого обхода. Слои отчета: Встроенные template-правила, Активный профиль, Итоговый runtime, Конфликты. Приоритет правил: template > профиль > catch-all. Дай краткую оценку UX и 3-5 конкретных улучшений для макета.'
```

## Выводы для макета

- Техническая структура с разделением источников правил полезна, но четыре слоя могут перегрузить пользователя.
- Заголовок `Исключения прямого обхода` лучше заменить на более короткий и понятный `Прямые маршруты`.
- Смысл обхода VPN лучше раскрывать подзаголовком: `Трафик, который идёт напрямую, минуя VPN`.
- Приоритет `template > профиль > catch-all` стоит показать визуально как последовательность принятия решения, а не только текстом.
- Конфликты лучше показывать рядом с итоговым правилом, а не уводить в отдельный изолированный блок.
- Полезно добавить диагностическое поле проверки конкретного домена или IP: пользователь вводит адрес и видит, какое правило победило.

## Решение для следующего UI-макета

- Заголовок блока: `Прямые маршруты`.
- Подзаголовок: `Адреса и группы, которые идут напрямую, минуя VPN`.
- Отдельные группы сохранить, но сделать итоговый runtime главным, а template/profile — раскрываемыми источниками.
- Конфликты показывать inline в строках итогового runtime и дублировать счётчик в сводке.
