## Цель

Перепроектировать основной web UI так, чтобы главное окно было сосредоточено на подключении и выборе узла, а маршрутизация, логи и прочие служебные вещи были вынесены в отдельные разделы через sidebar-навигацию.

## Границы

### Входит

- новый shell web UI с явным sidebar-меню;
- вынос маршрутизации и логов из главного рабочего экрана;
- перепроектирование иерархии topbar, действий и рабочих панелей;
- обновление JS-переключения view и тестового контракта `main_gui.html`.

### Не входит

- native GTK4 UI;
- смена backend API, кроме минимальных UI-зависимых правок;
- переписывание store, runtime, routing-модели или launcher-процесса.

## Контекст

- источник постановки: пользователь подтвердил, что browser fallback рабочий, и явно попросил перепроектировать UI;
- связанная бизнес-область: operational UX основного web GUI;
- ограничения и зависимости: главное окно должно оставаться рабочим, компактным и фокусироваться на подключении; служебные разделы нельзя потерять, их нужно унести в меню;

## Итог

`gui/main_gui.html` перепроектирован в двухзонный shell: слева появился `sidebar` с разделами `Подключение`, `Маршрутизация` и `Лог`, а справа остался основной рабочий canvas. Главное окно теперь держит только статус подключения, подписки и узлы; маршрутизация и лог убраны из постоянной главной компоновки и открываются как отдельные service-view через меню.

Клиентский JS обновлён под view-navigation: добавлено состояние текущего раздела, синхронизация заголовка рабочей области и служебных nav-meta/alert-сигналов для routing и logs. В `gui/gui_contract.py` обновлена версия GUI-контракта, а `tests/test_gui_server.py` переведены на новый layout-контракт. Локально подтверждены `python3 -m unittest tests.test_gui_server`, `python3 -m py_compile gui/gui_server.py gui/gui_contract.py` и `node --check` для извлечённого JS блока. Дальше нужна только пользовательская оценка удобства нового shell в browser mode.
