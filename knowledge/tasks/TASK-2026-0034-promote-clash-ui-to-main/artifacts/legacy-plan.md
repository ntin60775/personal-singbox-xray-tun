# Перевод Clash-кандидата в основной web UI

- Дата: 2026-04-02
- Статус: done
- Источник: подтверждение пользователя зафиксировать текущий Clash-кандидат как основной UI и дальше дорабатывать уже рабочую основную версию

## Цель

Сделать новый Clash-ориентированный экран основным интерфейсом маршрута `/`, при этом сохранить прежний встроенный UI на отдельном legacy-маршруте как fallback на время дальнейшей доработки.

## Изменения

- Переключить основной маршрут `/` и `/index.html` на новый asset `gui/design_review.html`.
- Оставить старый встроенный HTML `INDEX_HTML` доступным по отдельному legacy-маршруту.
- Обновить тесты так, чтобы основной asset был явно зафиксирован как текущий основной UI.
- Не трогать backend API и action endpoints: визуальный слой меняется, HTTP-контур управления остаётся тем же.

## Проверки

- `python3 -m py_compile gui/gui_server.py`
- `python3 -m unittest tests.test_gui_server`
- Локальный HTTP smoke:
  - открыть `/`;
  - открыть legacy-маршрут;
  - убедиться, что оба экрана отдаются.

## Допущения

- На этой итерации пользователь осознанно согласился сделать новый экран основным, даже если часть дальнейшей функциональной шлифовки ещё впереди.
- Legacy-маршрут нужен как безопасный fallback, но не должен рекламироваться как основной интерфейс.

## Итог

- Основной маршрут `/` и `/index.html` переведён на новый Clash-ориентированный экран из `gui/design_review.html`.
- Старый встроенный интерфейс сохранён как fallback на legacy-маршрутах:
  - `/legacy-ui`
  - `/legacy-ui.html`
  - `/classic-ui`
  - `/classic-ui.html`
- Добавлены явные константы backend для основного asset и legacy-маршрутов, чтобы дальнейшая доработка шла уже вокруг нового основного UI.
- Обновлены unit-тесты:
  - проверка основного asset;
  - проверка наличия legacy-маршрутов.
- Подтверждено:
  - `python3 -m py_compile gui/gui_server.py`;
  - `python3 -m unittest tests.test_gui_server`;
  - HTTP smoke на `/` и `/legacy-ui`.
