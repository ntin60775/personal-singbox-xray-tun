# Решения

Фиксируются ключевые архитектурные решения в ходе рефакторинга.

## 2026-06-09: Стек для доменной модели

**Решение:** Использовать `@dataclass` (Python 3.11+) для сущностей и value objects. Не тянуть Pydantic, attrs, или ORM.

**Причина:** Проект уже на Python 3.11+, `dataclass` — стандартная библиотека, достаточна для типизации и фабрик. Pydantic избыточен для in-memory модели.

## 2026-06-09: Репозитории — обёртка, не переписывание

**Решение:** Новые `JsonNodeRepository` и др. будут обёртками над существующими функциями `subvost_store`. Старые функции не удаляем — они остаются как внутренняя имплементация.

**Причина:** Минимизирует риск регрессий. Store — 1480 строк протестированного кода, переписывать с нуля опасно.

## 2026-06-09: Инкрементальный вывод из эксплуатации God Object

**Решение:** `SubvostAppService` остаётся working в течение всего рефакторинга. Методы переносятся в Use Cases по одному, с тестами на каждом шаге. В конце `SubvostAppService` становится thin orchestration layer (~400 строк).

**Причина:** Нельзя останавливать разработку на месяц. TUI должен работать после каждого коммита.
