# SDD: Аудит функционального разрыва TUI vs Legacy UI (TASK-2026-0064.1)

## Методология аудита

Источники истины:
1. `gui/gui_server.py` — все HTTP API endpoints (Web-GUI backend)
2. `gui/native_shell_app.py` — все экраны и действия GTK4 Native Shell
3. `gui/tui_app.py` — текущая реализация TUI

Критерий паритета: TUI должен предоставлять каждое действие, которое было доступно в старом UI, либо напрямую в интерфейсе, либо через CLI fallback с чёткой инструкцией.

## Раздел 5: Приоритезация доработок

### P0 (Критично — блокирует основной сценарий)

1. **Активация узла** (`/api/selection/activate`) — без этого нельзя выбрать ноду для подключения
2. **Импорт подписки по URL** (`/api/subscriptions/add`) — без этого нельзя добавить подписку
3. **Импорт отдельных ссылок** (`/api/import/preview` + `/api/import/save`) — без этого нельзя добавить ноду вручную

### P1 (Важно — основной UX)

4. **Ping узла** (`/api/nodes/ping`) — нужен для выбора лучшей ноды
5. **Удаление подписки** (`/api/subscriptions/delete`) — управление списком
6. **Обновление отдельной подписки** (`/api/subscriptions/refresh`) — не всегда нужно обновлять все
7. **Активация routing-профиля** (`/api/routing/activate`) — управление маршрутизацией
8. **Вкл/выкл маршрутизации** (`/api/routing/toggle`) — быстрое отключение routing

### P2 (Желательно — полный паритет)

9. **Импорт routing-профиля** (`/api/routing/import`) — редкая операция
10. **Обновление geodata** (`/api/routing/prepare-geodata`) — редкая операция
11. **Управление профилем** (`/api/profiles/update`, `/api/profiles/delete`) — переименование, удаление
12. **Управление узлом** (`/api/nodes/update`, `/api/nodes/delete`) — переименование, удаление
13. **Сброс активного routing** (`/api/routing/clear-active`) — редкая операция
14. **Вкл/выкл профиля routing** (`/api/routing/profile/update`) — редкая операция
15. **Terminate с confirm** — закрытие TUI с диалогом "Остановить VPN?"

## Раздел 6: Рекомендации по реализации

### Модальные диалоги (textual)

Для P0 и P1 нужны следующие ModalScreen:

1. `ImportSubscriptionModal` — поля: Название, URL, кнопки Добавить/Отмена
2. `ImportLinkModal` — TextArea для вставки ссылок, кнопки Импортировать/Отмена
3. `ConfirmModal` — уже есть, нужно расширить для "Активировать узел?"
4. `NodeContextMenu` — контекстное меню на строке таблицы (Активировать, Пинг, Удалить)

### DataTable row actions

textual DataTable поддерживает `cursor_type="row"` + событие `on_data_table_row_selected`. Для контекстного меню можно использовать:
- `on_data_table_row_selected` для активации
- Отдельный `Button` рядом с таблицей для действий над выбранной строкой

### Routing-управление

Добавить кнопки в RoutingTab:
- `Активировать` — для выбранного профиля
- `Включить/Выключить` — глобальный toggle routing
- `Импорт профиля` — диалог вставки JSON
- `Обновить geodata` — реальный вызов, не заглушка
