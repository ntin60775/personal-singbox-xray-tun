## Цель

Провести полный аудит функционального разрыва между старым UI (Web-GUI + GTK4 Native Shell) и новым TUI. Зафиксировать каждую недостающую функцию, заглушку и упрощение. Подготовить план доработки TUI до паритета со старым UI.

## Границы

### Входит

- Полный перечень API endpoints `gui_server.py` и их реализации в TUI
- Полный перечень экранов `native_shell_app.py` и их реализации в TUI
- Перечень заглушек (`self.notify("... пока через CLI")`) в `tui_app.py`
- Перечень действий, доступных в tray old-UI vs TUI tray
- SDD с приоритезацией доработок

### Не входит

- Собственно реализация доработок (это следующие подзадачи)
- Изменение Windows-части
- Изменение `gui_server.py` (он остаётся для совместимости, но не является целевым путём)
