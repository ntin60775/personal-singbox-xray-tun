## Цель

Довести новый `GTK4` UI-контракт до проверяемого состояния под фиксированное окно `1280×960`: убрать sidebar и banner, перевести shell на `Gtk.HeaderBar + Gtk.StackSwitcher`, переработать вкладку `Подключение` и сохранить `TASK-2026-0053.6` как запасной вариант.

## Границы

### Входит

- новый shell-контракт под фиксированный размер `1280×960`;
- горизонтальные вкладки в `Gtk.HeaderBar` вместо sidebar-навигации;
- удаление shell-баннера и перенос всей status-информации на вкладку `Подключение`;
- экран `Подключение` с одной status-row, action-row, `Gtk.InfoBar` при конфликте и отдельными блоками `Трафик` / `Интерфейс`;
- компактный экран `Узлы и подписки` с отдельными зонами подписок, узлов и маршрутизации;
- разгруженный экран `Диагностика` с разделением сводки и журнала и карточками summary-блоков;
- общая чистка UX-ошибок: повторов, жаргона, неинформативных кнопок, неадаптивных рядов действий;
- финальная полировка layout, списков, панелей действий и визуальной иерархии под `1280×960`.

### Не входит

- удаление fallback-кандидата `TASK-2026-0053.6`;
- массовые runtime/routing изменения вне прямой необходимости для нового UI;
- обязательная поддержка мобильных жестов в первом проходе;
- перевод нового дизайна в production launcher до ручного smoke и отдельного решения.

## Планируемые изменения

### Knowledge и разделение контуров

- сохранить `TASK-2026-0053.6` как резервный `GTK4`-вариант без потери его статуса и артефактов;
- использовать `TASK-2026-0054` как единственный контур нового редизайна;
- вести кодовую реализацию в ветке `task/task-2026-0054-full-ui-redesign`;
- синхронизировать `task.md`, `plan.md` и `registry.md` по мере продвижения.

### Глобальная перестройка навигации и компоновки

- использовать фиксированное окно `1280×960` без сценариев ресайза под меньшие размеры;
- собрать shell вокруг `Gtk.HeaderBar` с уменьшенной высотой;
- разместить `Gtk.StackSwitcher` в `HeaderBar` как основной переключатель вкладок;
- удалить sidebar и внешний информационный баннер;
- оставить в клиентской области только активную страницу `Gtk.Stack`.

### Экран `Подключение`

- собрать status-row из состояния подключения, выбранного узла, скорости `RX/TX` и `TUN`;
- сделать короткий action-row:
  - `Подключиться` или `Отключить`;
  - `Диагностика`;
  - `Узлы`;
- показывать конфликт другого экземпляра только через один `Gtk.InfoBar`;
- не прятать главное действие: кнопка `Подключиться` должна оставаться в интерфейсе даже при `sensitive=False`;
- сохранить быстрый переход в `Узлы` прямо с вкладки `Подключение`, не полагаясь только на верхнюю навигацию;
- вынести трафик и интерфейсные детали в отдельные компактные блоки;
- под фиксированное окно `1280×960` сразу показывать полный список DNS внутри блока `Интерфейс`, если он визуально помещается без деградации;
- добавить к строке активного узла контекст подписки;
- собрать дополнительные признаки (`tun`, логирование, маршрут) в отдельную строку статусов с symbolic-иконками, без пустых чекбоксов;
- оставить все служебные и результатные сообщения внутри вкладки `Подключение`, а не в shell-баннере.

### Экран `Узлы и подписки`

- использовать широкую двухколоночную композицию:
  - подписки слева;
  - узлы и маршрутизация справа;
- сократить строку подписки до домена, индикатора актуальности, действия обновления и overflow-меню;
- длинные URL обрезать с многоточием и показывать полностью во всплывающей подсказке;
- строку узла оставить компактной, но уже без жёсткой экономии ширины:
  - имя узла;
  - статус готовности;
  - компактное действие проверки пинга;
- выбор узла перенести на клик по строке, а не на отдельные тяжёлые кнопки;
- блок маршрутизации свернуть в `Gtk.Expander` и оставить в свёрнутом состоянии краткий статус активного профиля.

### Экран `Диагностика`

- разделить экран на два внутренних режима:
  - `Сводка`;
  - `Журнал`;
- на `Сводке` оставить только действительно нужные группы:
  - состояние;
  - параметры текущего подключения;
  - файлы и дампы;
  - экземпляры при конфликте;
- лишний технический текст и повторяющиеся пояснения убрать;
- журнал по умолчанию держать в отдельной вкладке, не смешивая его с summary-контекстом;
- в журнале оставить действия `Экспорт` и `Очистить`;
- убрать из представления по умолчанию шумные developer-oriented записи, если они не несут ценности пользователю.

### Общие UX-исправления

- разделить статус и действия: статус живёт внутри вкладки `Подключение`, действия — в компактном ряду под ним;
- не оставлять на главном экране технический жаргон, если он не нужен для немедленного решения пользователя;
- для долгих или асинхронных действий добавлять явную индикацию:
  - спиннер на `ping`;
  - disabled-state на недоступных действиях;
  - tooltip для сокращённых имён, URL и кнопок;
- полировать экран уже под штатный размер `1280×960`, не добавляя специальную логику для меньших размеров;
- уменьшить высоту `HeaderBar` через CSS и не держать в нём лишние подзаголовки.

### GTK4-технические ориентиры

- базовый размер окна: `set_default_size(1280, 960)` и фиксированный desktop-режим;
- основной shell строить на `Gtk.HeaderBar`, `Gtk.StackSwitcher`, `Gtk.Stack`, `Gtk.ScrolledWindow`, `Gtk.Box`, `Gtk.InfoBar`;
- для списков подписок и узлов использовать `Gtk.ListBox` с компактными `Gtk.ListBoxRow`;
- журнал держать в `Gtk.TextView` внутри `Gtk.ScrolledWindow` с ограничением по высоте;
- использовать стандартные symbolic-иконки, где это улучшает плотность и читаемость;
- `HeaderBar` уплотнить через CSS: `headerbar { min-height: 36px; padding: 0 8px; }`.

## Риски и зависимости

- русские подписи и длинные имена узлов всё ещё могут ломать layout, если не нормализовать обрезку и tooltip-контракт;
- попытка чрезмерно заполнить верхнюю status-line может снова превратить её в перегруженный banner;
- редизайн легко сорвётся в крупный diff без рабочего промежуточного состояния, если не идти vertical slice-ами;
- итоговая оценка остаётся зависимой от живого `GTK` smoke на реальном display.

- Внешний информационный banner удалён полностью: вся ключевая информация теперь находится во вкладке `Подключение`.
- Экран `Подключение` собран вокруг status-row, action-row, `Gtk.InfoBar` по конфликту экземпляров, блоков `Трафик` и `Интерфейс`, а также строки дополнительных статусов.
- После review defect-pass экран `Подключение` дополнительно очищен от дублей: конфликт чужого экземпляра теперь живёт только в одном `Gtk.InfoBar`, основная кнопка всегда остаётся видимой как `Подключиться` или `Отключить`, быстрый переход `Узлы` сохранён в action-row, строка активного узла дополнена подписью подписки, `TUN` убран из верхнего заголовка, а строка статусов переведена на `Gtk.Image` symbolic-иконки.
- DNS в блоке `Интерфейс` снова развёрнут под целевой размер `1280×960`: отдельная кнопка раскрытия скрыта, список адресов показывается сразу в карточке.
- Экран `Узлы и подписки` сохранён в широкой двухколоночной композиции: список подписок слева, узлы и маршрутизация справа. Compact row-контракты, tooltip-ы, icon-button обновления и popover-меню для подписок сохранены.
- Экран `Диагностика` разделён на внутренние режимы `Сводка` и `Журнал`: summary теперь живёт в отдельных карточках по группам, а лог оставлен в отдельной вкладке со старыми фильтрами, копированием и экспортом.
