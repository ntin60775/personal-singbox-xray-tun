## Цель

Подготовить репозиторий к безопасной публичной публикации без утечек локальных путей, операторских данных и персональных ссылок в `HEAD` и истории Git.

## Границы

### Входит

- аудит текущего дерева и истории Git;
- санитизация tracked-документации, desktop entry и конфиг-шаблонов;
- переписывание истории Git по тем же правилам;
- повторные проверки по поисковым маркерам и базовой локальной валидации.

### Не входит

- публикация на GitHub/GitLab;
- выбор лицензии вместо пользователя;
- ручной VPN-smoke с рабочей подпиской после обезличивания.

## Планируемые изменения

### Код

- убрать жёстко зашитые абсолютные пути из публичных desktop- и shell-артефактов;
- при необходимости ослабить provider-specific значения в tracked runtime-шаблоне.

### Конфигурация / схема данных / именуемые сущности

- заменить operator-specific `xray`-endpoint, домены и IP на безопасные placeholders или documentation-only примеры;
- задать набор правил санитизации для переписывания истории Git.

### Документация

- убрать из README и knowledge-артефактов приватные subscription URL, локальные пути и production-пути;
- зафиксировать остаточные риски публикации и отсутствие решения по лицензии, если оно не будет принято в этой сессии.

## Риски и зависимости

- автоматическая санитизация Markdown затрагивает большой исторический knowledge-контур, поэтому после rewrite обязательны целевые поисковые проверки по всей истории;
- force-push переписанной истории нужно делать осознанно и только после локальной валидации;
- до отдельного `git fetch origin --prune` локальный `origin/main` продолжает ссылаться на старую удалённую историю, даже если пользователь уже сделал force-push;
- без отдельного решения пользователя по лицензии репозиторий можно сделать публичным технически, но не полностью подготовленным юридически.

## Проверки

### Что можно проверить кодом или тестами

- `git grep` и `rg` по списку приватных маркеров в `HEAD` и по всей истории;
- `bash -n *.sh`
- `bash -n libexec/*.sh`
- `bash -n lib/*.sh`
- `python3 -m py_compile gui/gui_server.py gui/subvost_runtime.py gui/subvost_store.py gui/subvost_parser.py gui/embedded_webview.py`
- `python3 -m unittest tests.test_subvost_parser tests.test_subvost_store tests.test_subvost_runtime tests.test_gui_server tests.test_embedded_webview`
- `python3 ~/.agents/skills/markdown-localization-guard/scripts/markdown_localization_guard.py ...`

### Что остаётся на ручную проверку

- выбор лицензии перед публикацией;
- force-push переписанной истории в удалённый origin;
- `git fetch origin --prune` после force-push для синхронизации local remote-tracking refs;
- финальный human-review README и public positioning.
