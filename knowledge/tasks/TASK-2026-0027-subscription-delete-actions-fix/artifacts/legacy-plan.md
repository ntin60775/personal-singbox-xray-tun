# Исправление удаления подписок и row-actions в GUI

- Дата: 2026-04-01
- Статус: done
- Источник: запрос пользователя про некорректное удаление подписки `https://example.com/subscription`

## Цель

Исправить GUI-контракт действий по строкам подписок, профилей и узлов так, чтобы удаление подписки и остальные действия, завязанные на `dataset`, снова работали в браузере.

## Изменения

- Проверить генерацию `data-*`-атрибутов в helper-кнопках GUI и устранить несовместимость между camelCase-ключами и HTML-атрибутами.
- Обеспечить корректную передачу `subscriptionId`, `profileId` и `nodeId` в обработчики действий без ручного переписывания каждого сценария.
- Добавить регрессионную проверку, которая фиксирует ожидаемый DOM-контракт для action-кнопок.

## Проверки

- `python3 -m unittest tests.test_gui_server tests.test_subvost_store`
- `python3 -m py_compile gui/gui_server.py`
- Ручной GUI smoke без `Старт/Стоп`:
- добавить подписку по URL пользователя;
- удалить её кнопкой `Удалить`;
- проверить, что после ответа API запись исчезает из списка, а связанный профиль удаляется вместе с ней.

## Допущения

- Дефект находится на клиентской стороне GUI, потому что backend-функция `handle_subscription_delete()` удаляет подписку корректно при прямом вызове.
- Исправление должно быть общим для всех row-actions, а не только для удаления подписок, потому что один и тот же helper формирует атрибуты для профилей и узлов.

## Итог

- Исправлен helper `tinyButton()` в [gui_server.py](/repo/gui/gui_server.py): camelCase-ключи атрибутов теперь автоматически переводятся в kebab-case перед рендерингом `data-*`, поэтому `button.dataset.subscriptionId`, `profileId` и `nodeId` снова читаются браузером корректно.
- Из-за общего helper автоматически починились все row-actions, завязанные на `dataset`: удаление, переименование, включение/отключение и открытие профилей/узлов.
- Добавлен регрессионный тест в [test_gui_server.py](/repo/tests/test_gui_server.py), который фиксирует новый HTML/JS-контракт helper-а.
- Пройдены проверки: `python3 -m unittest tests.test_gui_server tests.test_subvost_store` и `python3 -m py_compile gui/gui_server.py`.
- Ручной браузерный smoke с реальным GUI в этой сессии не запускался; для пользовательского сценария стоит отдельно проверить удаление подписки `https://example.com/subscription` через интерфейс после перезапуска backend.
