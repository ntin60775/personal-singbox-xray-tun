# Импорт и обновление подписок по URL

- Дата: 2026-03-31
- Статус: done
- Источник: запрос пользователя автоматизировать обновление подписки вместо ручной правки конфига

## Цель

Сделать локально поддерживаемый сценарий добавления subscription URL и обновления набора узлов по кнопке, чтобы bundle мог сам подтягивать актуальный список нод и сохранять его в operator-managed store.

## Изменения

- Ввести локальное хранилище подписок:
  - display name;
  - URL подписки;
  - время последнего успешного обновления;
  - метаданные ошибки последнего обновления;
  - ETag / Last-Modified, если upstream это поддерживает.
- Реализовать fetch/update слой в Python:
  - ручное обновление одной подписки;
  - ручное обновление всех подписок;
  - понятное состояние `успешно / частично / ошибка`.
- Поддержать основные форматы ответа:
  - base64-encoded список ссылок;
  - plain-text список ссылок построчно;
  - fallback-отказ с диагностикой, если upstream-format не распознан.
- После обновления переводить контент подписки в ту же нормализованную модель узлов, что и импорт одиночных ссылок.
- Ввести dedup между узлами:
  - по стабильному fingerprint;
  - без молчаливой потери пользовательских меток и ручных переименований.
- Добавить GUI-действия:
  - `Добавить подписку`;
  - `Обновить`;
  - `Обновить все`;
  - `Показать ошибку обновления`;
  - `Отключить / удалить подписку`.

## Проверки

- Smoke с санитизированной тестовой подпиской, которая возвращает несколько ссылок.
- Негативные сценарии: таймаут, `404`, битый base64, пустой ответ, дублирующиеся ноды, частично невалидные строки.
- Ручной smoke: обновить подписку, увидеть новые узлы в UI, выбрать активный узел без правки `xray-tun-subvost.json`.

## Допущения

- В первой итерации не нужен фоновый auto-refresh daemon; достаточно явного обновления пользователем.
- Subscription URL остаётся локальными operator-managed данными и не попадает в Git, планы и README.
- Этот черновик вместе с импортом одиночных ссылок образует первую обязательную волну доработок.

## Итог

- Добавлено локальное хранилище подписок с `display name`, URL, статусом последнего обновления, `ETag` и `Last-Modified`.
- Реализованы ручное обновление одной подписки и `Обновить все`, распознавание `plain-text` и `base64`-ответов, а также сохранение последней ошибки обновления в store и UI.
- Обновление подписки переводит ссылки в общую нормализованную модель узлов, выполняет dedup по fingerprint и сохраняет пользовательские переименования узлов.
- Локально подтверждены unit-тесты и статические проверки; ручной smoke с реальной сетевой подпиской в этой сессии не выполнялся и остаётся остаточным риском.
