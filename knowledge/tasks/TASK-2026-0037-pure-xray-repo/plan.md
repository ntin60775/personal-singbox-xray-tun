## Цель

Превратить текущий bundle в чистый `xray`-репозиторий без рабочего `sing-box`-контура, без builtin-фоллбэков и без авто-миграции из tracked-шаблона, а также отдельно убедиться, что импорт подписок по URL остаётся рабочим для указанной пользователем ссылки.

## Границы

### Входит

- удаление рабочего `sing-box` runtime из кода и tracked-конфигов;
- перенос основного запуска на чистый `xray`;
- сохранение и проверка import/refresh подписок по URL;
- обновление GUI, diagnostics, installer и документации.

### Не входит

- полная редактура всех исторических knowledge-артефактов;
- rename каталога репозитория на уровне ФС;
- скрытые backup-файлы и дополнительные запасные runtime-контуры.

## Планируемые изменения

### Код

- сделать основной `run-xray-tun-subvost.sh` чистым `xray` runtime;
- убрать `sing-box` из stop/diagnostics/installer/GUI-status;
- слить временный `xray-only` контур в основной runtime и убрать дублирующие wrapper'ы;
- оставить в web-контуре один основной HTML asset и убрать отдельную встроенную версию GUI из backend-а;
- убедиться, что parser/store/import подписок остаётся рабочим на реальном payload.
- если провайдер возвращает stub вместо нод, показывать точную причину и не терять real-user контекст для `HWID` при root-backed GUI.

### Конфигурация / схема данных / именуемые сущности

- оставить один основной tracked runtime-конфиг на `xray`;
- удалить рабочие tracked-артефакты `sing-box`;
- упростить state-модель до одного runtime и убрать builtin/runtime-preference ветки.

### Документация

- переписать README и AGENTS под чистый `xray` runtime;
- зафиксировать задачу и итог в knowledge-системе.

## Риски и зависимости

- часть shell/Python-логики сейчас уже раздвоена между `stack` и `xray-only`, её нужно аккуратно схлопнуть без регрессии;
- конкретная подписка пользователя может использовать формат или содержимое, которое потребует точечной доработки parser/store;
- физические удаления файлов подпадают под отдельное правило показа точного списка и `COUNT`.

## Критерии завершения

- основной runtime репозитория не зависит от `sing-box`;
- старт bundle возможен только из явно выбранного активного узла;
- import/refresh подписок по URL подтверждён на пользовательской ссылке;
- рабочие `sing-box`-артефакты и их code-path удалены;
- выполнены доступные автоматические проверки и localization guard;
- ручной smoke явно отмечен как отдельная оставшаяся проверка, если среда не позволяет сделать его здесь.
