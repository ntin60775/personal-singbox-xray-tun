## Цель

Полностью убрать `sing-box` из текущего репозитория и привести bundle к чистому `xray`-проекту без двойного runtime-контура, без builtin-фоллбэков и без авто-миграции из tracked-шаблона, при этом сохранить рабочий импорт подписок по URL и отдельно подтвердить поддержку подписки `https://example.com/subscription`.

Ожидаемый результат: основной runtime, GUI, installer, диагностика, документация и tracked-конфиги опираются только на `xray-core`, в репозитории не остаётся рабочих зависимостей от `sing-box`, а запуск bundle возможен только из явно выбранного активного узла.

## Текущий этап

Кодовая миграция завершена: основной runtime, GUI, installer, tracked-конфиг и parser/store уже переведены на чистый `xray`, legacy-файлы удалены, а для пользовательской подписки подтвержден рабочий payload при `Xray`-совместимом наборе заголовков. Дополнительно убраны builtin-фоллбэк и авто-миграция из `xray-tun-subvost.json`, а web-GUI схлопнута в один основной рабочий экран на базе `gui/main_gui.html` без отдельной встроенной версии в backend.

На финальной полировке subscription-flow выявилась отдельная регрессия UX: при provider-stub backend показывал слишком общую ошибку про несовместимый клиент, хотя реальный ответ провайдера был `Достигнут лимит устройств`. Это исправлено: parser теперь поднимает точный provider-message, а `HWID` для GUI-запросов через `pkexec` рассчитывается из real-user home, а не из контекста `root`. После ручной очистки лишних устройств у провайдера живая проверка пользовательской ссылки снова дала base64-подписку с 6 рабочими нодами.

Поверх этого выделена и закрыта подзадача `TASK-2026-0037.1`: в основном web-интерфейсе убраны лишние декоративные блоки sidebar, а боковое меню перестало быть статичным и теперь ведёт к рабочим секциям текущего single-screen layout.

Следом закрыта и подзадача `TASK-2026-0037.2`: постоянная боковая панель убрана полностью, источники и узлы собраны в один операционный блок, ручной импорт схлопнут в раскрываемую секцию, а дополнительные UI-маршруты удалены так, чтобы у GUI осталась одна каноническая точка входа.

После этого закрыта подзадача `TASK-2026-0037.3`: основной экран ужат до fullscreen operational UI без вертикальной прокрутки страницы, backend получил живой ping-эндпоинт и базовые счётчики/скорости по `tun`, а главный экран оставил только runtime-действия, подписки, узлы и постоянный лог с явными ошибками.

Поверх этого закрыта подзадача `TASK-2026-0037.4`: верхняя action-зона получила более плотный operational toolbar с одинаковыми по ширине кнопками `Старт` / `Стоп` / `Диагностика`, runtime начал сохранять `STARTED_AT` в state-файл для отображения времени текущего подключения, а баннер последней ошибки получил локальный dismiss с автоматическим повторным показом при новой ошибке.

Финальный технический follow-up по этой же задаче уточнил стартовый shell-контур и tracked-артефакты: если локальный store уже существует, `run-xray-tun-subvost.sh` перед preflight-валидацией заново синхронизирует `generated-xray-config.json` из store, чтобы старт опирался на актуальный snapshot активной ноды без возврата к авто-миграции из tracked-шаблона. Параллельно tracked `subvost-xray-tun.desktop` сохранён в portable-виде с `Icon=network-vpn`, а самосинхронизация абсолютного SVG-пути остаётся обязанностью runtime/helper-цепочки bundle, а не репозиторного launcher-файла. Для tracked `xray-tun-subvost.json` отдельно зафиксирован репозиторный DoH fallback `https://8.8.8.8/dns-query` и добавлен unit-регресс на это значение.

## Итог

Основной runtime переведён на чистый `xray`: `run-xray-tun-subvost.sh`, `stop-xray-tun-subvost.sh`, diagnostics, installer, GUI-status и tracked `xray-tun-subvost.json` больше не зависят от второго proxy-движка. Из корня и `libexec/` удалены legacy-файлы `singbox-tun-subvost.json`, `run-xray-only-tun-subvost.sh`, `stop-xray-only-tun-subvost.sh`, `libexec/run-xray-only-tun-subvost.sh`, `xray-only-tun-subvost.json`.

Поверх базовой миграции убраны остатки запасного режима: store больше не мигрирует live-узел из tracked-шаблона, GUI не умеет переключать runtime на builtin-config, а стартовый shell-скрипт требует явный активный узел и `generated-xray-config.json`. Если активный узел отсутствует или стал невалидным, bundle больше ничего не подбирает автоматически и блокирует следующий старт до явного выбора ноды.

Подписочный flow сохранён и уточнён: refresh теперь отправляет `Xray`-совместимые заголовки (`User-Agent`, `X-HWID`, `X-Device-OS`, `X-Ver-OS`, `X-Device-Model`), parser отклоняет провайдерские заглушки и поддерживает `VLESS xhttp extra=...`. На живой ссылке `https://example.com/subscription` локально подтвержден base64-payload с шестью рабочими нодами, который разбирается текущим parser-ом.

Поверх этого исправлен диагностический слой subscription-flow: если провайдер отдаёт stub-ссылку с конкретной причиной, GUI теперь показывает именно её, например `Достигнут лимит устройств`, вместо прежней размытой формулировки. Для запросов из root-backed GUI `HWID` теперь строится из `SUBVOST_REAL_HOME`, чтобы не плодить лишние “устройства” из-за запуска backend-а через `pkexec`.

Отдельно дочищен web-контур: основной GUI asset переименован в `gui/main_gui.html`, в `gui_server.py` больше нет отдельного встроенного HTML-экрана и дополнительных UI-маршрутов, а сама страница сведена к одному компактному fullscreen operational layout. Новая страница реально работает с текущими API backend-а: добавляет и обновляет подписки по URL, показывает узлы выбранной подписки, сразу активирует узел по клику строки, умеет делать ping узла, показывает live-метрики трафика и скорости по `tun`, а также ведёт единый видимый лог действий и ошибок.

Финальный follow-up по скриншотному UX-фидбеку тоже закрыт: toolbar теперь сразу показывает длительность активного подключения по реальному времени старта runtime, action-кнопки выровнены по ширине и стали заметно удобнее для клика, а баннер последней ошибки можно скрыть без очистки самого лога. При появлении новой ошибки баннер открывается снова, поэтому явная индикация проблем не теряется.

Последний технический проход добил три оставшихся острых угла. Во-первых, стартовый runtime теперь при наличии локального store пересобирает `generated-xray-config.json` прямо перед проверкой активной ноды, поэтому bundle использует свежий snapshot без возврата к старой миграции из tracked `xray-tun-subvost.json`. Во-вторых, репозиторный `subvost-xray-tun.desktop` снова оставлен portable: tracked-файл хранит `Icon=network-vpn`, а absolute path к `assets/subvost-xray-tun-icon.svg` по-прежнему материализуется только локальной helper-цепочкой bundle. В-третьих, в tracked-шаблоне зафиксирован DoH fallback `https://8.8.8.8/dns-query`, и это прикрыто unit-тестом, чтобы значение не съехало в следующих правках незаметно.

Остаточный риск: полный `sudo` smoke runtime, проверка `tun0`, `ip rule` и ручной GUI-поток реального старта/остановки не выполнялись в этой сессии и должны быть подтверждены отдельно на целевой машине.
