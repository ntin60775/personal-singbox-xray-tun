## Цель

Закрыть пользовательский feedback round по верхней панели и баннеру ошибок, не перестраивая весь fullscreen UI заново.

## Границы

### Входит

- одинаковая ширина и увеличенная высота action-кнопок;
- отображение времени подключения в header/topbar;
- dismiss для последней ошибки;
- минимальное расширение shell/backend-state под таймер.

### Не входит

- новые backend endpoints;
- переработка структуры логов;
- любые дополнительные сетевые или installer-изменения.

## Планируемые изменения

### Код

- пробрасывать это время через `collect_status()` в payload GUI;
- добавить в header отдельный metric/context chip для времени подключения;
- сделать action-row из трёх одинаковых по ширине кнопок с большей высотой;
- добавить кнопку скрытия баннера последней ошибки и логику повторного показа при новой ошибке.

### Конфигурация / схема данных / именуемые сущности

### Документация

## Риски и зависимости

- таймер не должен показывать ложные значения при частичном или деградированном состоянии runtime;
- dismiss баннера не должен прятать новую ошибку после следующего действия;
- выравнивание кнопок не должно ломать мобильный fallback layout.
