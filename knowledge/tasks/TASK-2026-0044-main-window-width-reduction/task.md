## Цель

Уменьшить стартовую ширину главного GUI-окна до `1280` px и синхронно сузить максимальную ширину основного web-интерфейса так, чтобы компоновка сохранила прежние внешние отступы и не получила очевидных регрессий.

## Границы

### Входит

- уменьшение номинальной ширины по умолчанию для встроенного `GTK/WebKitGTK` окна;
- уменьшение максимальной ширины основного контейнера `main_gui.html`;
- обновление unit-тестов на расчёт стартового размера окна;
- синхронизация task-артефактов и реестра задач.

### Не входит

- перепроектирование сетки и breakpoints интерфейса;
- ручная живая проверка GUI через реальное графическое окружение;
- изменения высоты окна, runtime-логики и backend API.

## Контекст

- источник постановки: пользователь попросил уменьшить ширину главного окна программы и отдельно уточнил, какая ширина используется сейчас по умолчанию;
- связанная бизнес-область: локальный desktop GUI bundle для `xray` TUN runtime;
- ограничения и зависимости: нужно синхронно скорректировать размер окна и web-контейнера, чтобы не сжать только оболочку без пересмотра внутренней компоновки;
- основной контекст сессии: `новая задача`.

- максимальная ширина основного web-контейнера уменьшена синхронно и не конфликтует с текущими внешними отступами;

## Итог

Стартовая ширина встроенного `GTK/WebKitGTK` окна уменьшена с `1440` до `1280` px. Максимальная ширина основного контейнера `.app-shell` синхронно уменьшена с `1360px` до `1200px`, чтобы сохранить прежнюю логику внешних отступов и не сузить только оболочку без изменения самого интерфейса.
