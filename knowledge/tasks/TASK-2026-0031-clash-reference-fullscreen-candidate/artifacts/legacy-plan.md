# Полноэкранный Clash-референс для web UI

- Дата: 2026-04-02
- Статус: done
- Источник: уточнение пользователя взять за основу Clash Verge Rev-style и доработать его под более светлую тёмную тему, полную локализацию, осмысленные метрики и отсутствие вертикального скролла в развёрнутом окне

## Цель

Собрать один основной кандидат web UI по мотивам `Clash Verge Rev`, который выглядит ближе к реальному desktop proxy-клиенту и в развёрнутом окне браузера укладывается по высоте экрана без необходимости прокручивать интерфейс вниз.

## Изменения

- Убрать stacked review-страницу с несколькими вариантами и заменить её на один Clash-ориентированный полноэкранный кандидат.
- Осветлить тёмную тему:
  - уйти от почти чёрного фона;
  - поднять читаемость панелей и текста;
  - сохранить характер desktop shell из семейства Clash.
- Полностью локализовать интерфейс на русский, оставляя английский только в именах внешних продуктов и машинно-значимых литералах.
- Заменить декоративные и текстовые блоки на более осмысленные operational-метрики:
  - состояние стека;
  - готовность следующего запуска;
  - покрытие профилей и узлов;
  - свежесть подписок;
  - диагностика и последнее действие.
- Переложить layout на фиксированный по высоте fullscreen-shell для развёрнутого окна:
  - главный каркас должен использовать высоту viewport;
  - приоритет на рост в ширину;
  - для более узких окон допускается обычная адаптация без отдельного масштабирования.

## Проверки

- `python3 -m py_compile gui/gui_server.py`
- `python3 -m unittest tests.test_gui_server`
- Локальный HTTP smoke:
  - открыть `/design-review`;
  - проверить, что рендерится один Clash-кандидат;
  - убедиться, что данные продолжают приходить из `/api/store`.

## Допущения

- На этой итерации меняется review-маршрут `/design-review`, а не основной production-маршрут `/`.
- Требование «без вертикального скролла» относится к развёрнутому окну браузера на desktop, а не к любому узкому или уменьшенному окну.
- Чтобы реально соблюдать ограничение по высоте, review-экран должен быть single-candidate, а не длинной страницей со стеком нескольких вариантов.

## Итог

- `gui/design_review.html` полностью пересобран из multi-variant review в один fullscreen-кандидат по мотивам `Clash Verge Rev`.
- В интерфейсе:
  - осветлена тёмная тема;
  - убрана смесь языков в пользовательских подписях;
  - добавлены осмысленные operational-метрики вместо длинных текстовых блоков;
  - главный layout переложен на высоту viewport для развёрнутого окна.
- Для большого desktop-окна добавлен режим с `100dvh` и `overflow: hidden`, чтобы основная поверхность укладывалась без вертикального скролла; для более узких экранов сохранена обычная адаптация.
- Обновлён unit-тест под новый single-candidate экран.
- Подтверждено:
  - `python3 -m py_compile gui/gui_server.py`;
  - `python3 -m unittest tests.test_gui_server`;
  - локальный HTTP smoke на `/design-review`.
- Остаточный риск: browser-level визуальная проверка реального fullscreen-рендера в этой сессии не автоматизировалась через Playwright; подтверждены HTML-отдача, структура layout и локальные проверки.
