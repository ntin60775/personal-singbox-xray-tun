# Убрать глобальный direct для национальных доменов

- Дата: 2026-03-28
- Статус: done
- Источник: пользовательская диагностика поведения `subvost-xray-tun` при включённом VPN

## Цель

Перевести runtime-policy на модель `default -> proxy`, убрав общее исключение `direct` для `.ru`, `.su` и `.xn--p1ai`, которое ломает часть доменов Яндекса после истечения DNS-кэша.

## Изменения

- Обновить `singbox-tun-subvost.json`:
  - убрать DNS-правило, которое отправляет `.ru/.su/.xn--p1ai` в `dns-direct`;
  - убрать route-правило, которое отправляет `.ru/.su/.xn--p1ai` в `direct`;
  - сохранить служебный `direct` только для процессов `xray` и `sing-box`.

## Проверки

- Проверить, что JSON остаётся валидным.
- Если доступен `sing-box`, выполнить `sing-box check` для обновлённого конфига.
- Зафиксировать, что после правки новые точечные обходы VPN должны добавляться отдельным allowlist.

## Допущения

- Список доменов, которым позже понадобится `direct`, пока неизвестен и будет добавлен отдельной задачей.
- Изменение не должно затрагивать `xray-tun-subvost.json`, потому что источник проблемы находится в policy/DNS-части `sing-box`.

## Итог

- В `singbox-tun-subvost.json` удалены общее DNS-правило и route-правило, которые отправляли `.ru/.su/.xn--p1ai` в `direct`.
- Итоговая active-policy теперь соответствует модели `default -> proxy`, а `direct` сохранён только для служебных процессов `xray` и `sing-box`.
- Проверки пройдены: `python3 -m json.tool` и `sing-box check -c /opt/subvost-xray-tun/singbox-tun-subvost.json`.
