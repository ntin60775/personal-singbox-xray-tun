# Планы проекта

Каталог `plans/` хранит инженерные планы задач, которые требуют больше одного очевидного изменения.

## Структура

- `drafts/` — черновики, где ещё может меняться объём и способ реализации
- `active/` — планы в работе или ожидающие реализации
- `done/` — завершённые или закрытые планы
- `cancelled/` — остановленные планы, которые решили не реализовывать

Эта структура каталогов фиксирована для репозитория. Планы перемещаются между `drafts/`, `active/`, `done/` и `cancelled/`, но сами каталоги сохраняются даже пустыми и не удаляются без отдельной явной команды.

## Именование

Используйте формат:

```text
YYYY-MM-DD-short-slug.md
```

Пример:

```text
2026-03-27-tun-hardening-from-report.md
```

## Минимальный шаблон

Каждый план должен содержать:

```markdown
# Короткий заголовок

- Дата: YYYY-MM-DD
- Статус: draft | planned | in_progress | done | cancelled
- Источник: ссылка на задачу, отчёт или файл

## Цель

Что именно должно измениться.

## Изменения

- Конкретные доработки по подсистемам.

## Проверки

- Локальные проверки и ручной smoke.

## Допущения

- Ограничения и принятые решения.

## Итог

Заполняется при завершении и кратко фиксирует факт реализации.
```

## Жизненный цикл

1. Новый нетривиальный task получает файл в `drafts/`, если объём ещё уточняется, либо сразу в `active/`, если задача уже понятна.
2. По ходу работы план обновляется, если меняется объём, риски или способ проверки.
3. Когда черновик стабилизирован, файл переносится из `drafts/` в `active/`.
4. После завершения задачи в план добавляется раздел `Итог`.
5. Затем файл переносится в `done/`.
6. Если реализация не нужна или решение признано неверным, в файл добавляется раздел `Причина отмены`, после чего он переносится в `cancelled/`.

Во всех сценариях выше перемещается именно файл плана. Каталоги состояний остаются на месте, даже если временно пусты.

## Чего здесь не должно быть

- секретов;
- рабочих JSON-конфигов с приватными адресами;
- сырых диагностических дампов из `logs/`;
- расплывчатых TODO без проверок и критериев завершения.
