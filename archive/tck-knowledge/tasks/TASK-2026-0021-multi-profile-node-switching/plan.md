## Исторический контекст

### Изменения

- Спроектировать локальную сущность `profile`:
  - источник: ручной импорт или подписка;
  - список узлов;
  - пользовательское имя;
  - флаги enabled/disabled;
  - история последнего выбора.
- Добавить сущность `active selection`:
  - активный профиль;
  - активный узел;
  - время последней активации;
  - источник выбора.
- Генерировать runtime-конфиг `xray-tun-subvost.json` из локальной модели, а не считать его primary source of truth.
- Сохранить совместимость с текущим Linux runtime:
  - запуск по-прежнему идёт через текущие shell-скрипты;
  - TUN/DNS/state flow не переписывается целиком на этом этапе.
- Добавить UI-операции:
  - список профилей;
  - список узлов внутри профиля;
  - `Сделать активным`;
  - `Переименовать`;
  - `Отключить`;
  - `Удалить`.
- Отдельно определить правила миграции с текущего single-config режима:
  - как bundle создаёт первый профиль из имеющегося локального конфига;
  - как не потерять operator-managed placeholders и приватные runtime-значения.

## Исторические проверки

- Ручной smoke: импортировать два узла, переключиться между ними и убедиться, что активный runtime-конфиг меняется без ручного редактирования JSON.
- Проверить restart bundle после выбора нового узла и корректное восстановление состояния.
- Проверить сценарий, когда один профиль или один узел удаляется, а активный выбор должен корректно переехать на fallback.

## Исторические допущения

- На первом этапе одновременно активен только один узел и один runtime-стек.
- Хранение профилей должно быть локальным и вне Git, даже если bundle physically лежит в Git-репозитории.
- Схема хранения должна одинаково обслуживать и будущий `GTK`, и возможный `Tauri 2`, чтобы не переписывать data/model слой второй раз.

## Исторический итог

- Введена локальная модель `profile + nodes + active selection` в `~/.config/subvost-xray-tun/store.json`; manual import использует профиль `Локальные ссылки`, а каждая подписка получает собственный профиль.
- Активный узел и профиль материализуются в локальный `generated-xray-config.json`; `state file` теперь хранит путь активного runtime-конфига и связанный active selection.
- GUI получил список профилей и узлов, операции активации, переименования, отключения и удаления, а также fallback active selection при исчезновении активного узла.
- Добавлена миграция с live single-config режима в первый локальный профиль, если store ещё не создан.
- Локально подтверждены unit-тесты и статические проверки; ручной runtime smoke со сменой активного узла и последующим живым restart не выполнялся и остаётся остаточным риском.
