# Архивный ручной smoke GTK4 UI от 2026-04-10

## Контекст

- источник: архивная Codex smoke-сессия для `GTK4` native shell;
- launcher: `./open-subvost-gtk-ui.sh`;
- дата smoke: `2026-04-10`;
- перенос в репозиторий выполнен позднее как восстановление потерянной подзадачи `TASK-2026-0053.6`.

## Среда и геометрия

- display: `2560x1440`;
- окно: `Subvost Xray TUN Native Shell`;
- фактическая geometry: `2892x1999+-12+-12`;
- `WM_NORMAL_HINTS` minimum size: `2892x1999`;
- `_NET_WM_ALLOWED_ACTIONS` включал `RESIZE`, `MAXIMIZE_HORZ` и `MAXIMIZE_VERT`, но фактическое поведение не соответствовало этим возможностям.

## Поведение resize и доступности

- попытка уменьшить окно до `1400x980` не изменила geometry: после команды размер остался `2892x1999`;
- архивный план трактовал это как фактически неработающий `resize/maximize`;
- в том же smoke окно `Settings` открывали через D-Bus control method, потому что правый край header-а частично уезжал за экран и часть `CTA` была недоступна напрямую.

## Повторяющиеся warnings

- в логах idle-сценария многократно появлялись предупреждения вида `GtkBox ... natural size must be >= min size`;
- эти warnings согласуются с тем, что shell сам объявлял minimum size больше физического экрана.

## Подтверждённые UX-дефекты

- default и minimum size окна больше экрана;
- `resize/maximize` фактически не работают;
- верхний banner и поясняющие тексты занимают полезную площадь;
- `StackSidebar` выбивается из тёмного контракта;
- `Log` выглядит как белая surface внутри тёмного shell;
- selector темы вводит в заблуждение и обещает режимы, которых фактически нет.
