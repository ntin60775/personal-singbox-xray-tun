# Атомарное обновление подписок и rollback на builtin

- Дата: 2026-04-01
- Статус: done
- Источник: запрос пользователя убрать хвост с `last_error`, запретить partial-refresh и добавить rollback на встроенный config при ошибках подписок

## Цель

Сделать сценарии добавления и обновления подписок строго атомарными: либо новая версия подписки полностью валидна и применяется целиком, либо bundle сохраняет предыдущую рабочую версию и переводит следующий старт в режим `builtin`.

## Изменения

- Исправить сохранение `last_error` после hard-error одиночного обновления подписки, чтобы ошибка переживала reload UI.
- Убрать `partial-refresh`:
  - если в payload есть хотя бы одна невалидная строка, обновление считается ошибкой;
  - рабочая версия узлов подписки не переписывается.
- Добавить rollback на `builtin` при ошибках подписок:
  - для существующей подписки сохраняется старая рабочая версия;
  - для новой подписки неуспешное первое обновление откатывает создание подписки целиком.
- Упростить backend/UI-статусы вокруг `refresh-all`, чтобы не использовать термин `partial` для одиночных подписок.
- Добавить тесты на атомарность обновления, rollback и persistence ошибок.

## Проверки

- `python3 -m unittest tests.test_subvost_store tests.test_gui_server`
- `python3 -m py_compile gui/gui_server.py`
- Безопасный HTTP smoke локального GUI backend без `Старт/Стоп`:
  - hard-error одиночного refresh сохраняет `last_error` и переводит `runtime_preference` в `builtin`;
  - invalid payload не меняет предыдущую рабочую версию подписки;
  - неуспешное первое обновление новой подписки не оставляет полусозданную подписку.

## Допущения

- Rollback на `builtin` означает только выбор источника runtime для следующего старта; текущий live-стек в рамках этой задачи не перезапускается.
- Дубликаты валидных строк не считаются ошибкой payload, но должны схлопываться в один узел без partial-статуса.

## Итог

- Хвост с `last_error` закрыт: hard-error одиночного refresh теперь переживает reload, потому что store сохраняется до возврата ошибки пользователю, а `runtime_preference` переключается в `builtin`.
- `partial-refresh` убран: если в payload подписки есть хотя бы одна строка с parse-error, обновление не применяется целиком, рабочая версия узлов остаётся прежней, а backend возвращает ошибку.
- Неуспешное первое обновление новой подписки стало атомарным: созданная подписка и её профиль удаляются, а bundle возвращается в режим `builtin`.
- Для существующей подписки при ошибке сохраняется старая рабочая версия, а rollback на `builtin` влияет только на следующий старт, не перезапуская текущий live-стек.
- Проверки пройдены: `python3 -m unittest tests.test_subvost_store tests.test_gui_server` и `python3 -m py_compile gui/gui_server.py`.
