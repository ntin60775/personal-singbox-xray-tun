## Цель

Перевести встроенный launcher в аппаратно-независимый устойчивый software-rendering режим на машинах, где `WebKitGTK` даёт белый экран из-за `GBM/DRM`-пути.

## Границы

### Входит

- fallback на аппаратно-независимый software-rendering через поддерживаемые `WEBKIT_*` переменные и `WebKit.Settings`;
- минимальная регрессия в unit-тестах;
- документирование фактической причины и результата.

### Не входит

- переписывание HTML/JS интерфейса;
- отдельная ветка только для детекта конкретного GPU-вендора;
- миграция на иной desktop runtime.

## Планируемые изменения

### Код

- добавить в launcher явный набор переменных окружения для аппаратно-независимого software-rendering пути `WebKitGTK`;
- дополнительно выключить аппаратное ускорение и `WebGL` на уровне `WebKit.Settings`;
- покрыть helper-функции unit-тестами.

### Конфигурация / схема данных / именуемые сущности

- сохранить текущий контракт `open-subvost-gui.sh` и `embedded_webview.py`, не вводя новый пользовательский workflow;
- приоритет отдать безопасному CPU/SHM-рендеру вместо GPU/DMABUF, не завязываясь на конкретного производителя графики.

### Документация

- завести task-артефакты `TASK-2026-0042`;
- обновить `knowledge/tasks/registry.md`;
- после завершения синхронизировать `Итог` и проверки.

## Риски и зависимости

- отключение ускорения может слегка ухудшить плавность, но это приемлемо для локального операционного GUI и даёт одинаковое поведение на разных GPU;
- на машинах, где `WebKitGTK` уже работает стабильно, fallback не должен ломать запуск;
- полноценное подтверждение исправления требует живого запуска на проблемной машине.

## Фактический результат

- `libexec/open-subvost-gui.sh` теперь запускает `embedded_webview.py` с аппаратно-независимыми `WEBKIT_*` defaults для CPU/SHM-рендера, но позволяет переопределить их извне.
- `gui/embedded_webview.py` дополнительно применяет те же defaults и выключает `hardware acceleration`, `WebGL` и ускорение `2d canvas` через `WebKit.Settings`, если конкретный runtime это умеет.
- `tests.test_embedded_webview` получил проверки helper-функций software-rendering режима, а `tests.test_gui_server` теперь страхует наличие нужных `WEBKIT_*` defaults в launcher-е.
- Локально подтверждены shell syntax, `py_compile`, unit-тесты и `python3 gui/embedded_webview.py --check`; живой desktop smoke остаётся последним ручным подтверждением.
