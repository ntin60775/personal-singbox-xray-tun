## Цель

Аккуратно заменить запуск GUI во внешнем браузере на встроенное `GTK/WebKitGTK` окно без смены backend-контракта и без полноценной миграции на `Tauri`.

## Границы

### Входит

- новый Python-модуль встроенного webview launcher-а;
- переключение shell launcher-а на новый модуль;
- installer-зависимости для `GTK/WebKitGTK`;
- unit/static проверки, которые можно добавить без живого GUI;
- task-centric документация по задаче.

### Не входит

- нативный `GTK`-rewrite интерфейса;
- перенос проекта на `Rust`/`Tauri`;
- расширение HTTP API backend-а без прямой необходимости для launcher-а.

## Планируемые изменения

### Код

- добавить Python-скрипт, который открывает встроенное окно, грузит `http://127.0.0.1:8421` и умеет fallback-иться на системный браузер;
- обновить `libexec/open-subvost-gui.sh`, чтобы backend по-прежнему стартовал так же, но GUI открывался через новый launcher;
- при необходимости скорректировать desktop entry и связанные helper-скрипты.

### Конфигурация / схема данных / именуемые сущности

- расширить `install-on-new-pc.sh` пакетами для `python3-gi`, `GTK` и `WebKitGTK`;
- зафиксировать устойчивую схему выбора `WebKit`-namespace для разных Linux-окружений.

### Документация

- создать task-артефакты `TASK-2026-0038`;
- синхронизировать `knowledge/tasks/registry.md`;
- отразить фактические проверки и остаточные риски после завершения.

## Риски и зависимости

- на разных Debian/Ubuntu-сборках могут отличаться доступные `WebKit` namespace и пакетные имена;
- GUI backend остаётся root-процессом через `pkexec`, а окно должно запускаться только от обычного пользователя;
- автоматический browser-level smoke ограничен, поэтому часть проверки останется ручной;
- fallback на браузер нужен, чтобы не превратить launcher в жёстко-завязанный на одно desktop-окружение.

## Проверки

### Что можно проверить кодом или тестами

- `bash -n *.sh`
- `bash -n libexec/*.sh`
- `bash -n lib/*.sh`
- `python3 -m py_compile gui/gui_server.py gui/subvost_runtime.py gui/subvost_store.py gui/subvost_parser.py gui/embedded_webview.py`
- `python3 -m unittest tests.test_gui_server tests.test_embedded_webview`
- `python3 gui/embedded_webview.py --check`

### Что остаётся на ручную проверку

- fallback на внешний браузер, если `WebKitGTK` недоступен.

## Фактический результат

- Добавлен модуль `gui/embedded_webview.py` с runtime-probe, выбором совместимого `GTK/WebKitGTK` стека и запуском встроенного окна через `Gtk.Application`.
- `libexec/open-subvost-gui.sh` теперь поддерживает `SUBVOST_GUI_LAUNCH_MODE=auto|webview|browser`, по умолчанию запускает встроенное окно и только потом делает fallback на `xdg-open`.
- `libexec/install-on-new-pc.sh` доустановит `python3-gi` и доступные GI-пакеты для desktop webview, не предполагая жёстко только `WebKit 6.0`.
- `gui/embedded_webview.py` теперь не ограничивается фиксированным `default_size`: launcher подбирает стартовую геометрию от размера монитора и открывает окно почти на всю доступную высоту, не раздувая его шире рабочего пространства.
- `gui/main_gui.html` дополнительно отполирован под desktop-webview: topbar переразложен в более читаемый control-header, действия и метрики получили более явную иерархию, а панели подписок, узлов и лога стали контрастнее и спокойнее в восприятии.
- Во втором проходе `gui/main_gui.html` переведён на щадящий DOM-update: header обновляется как прежде, но списки и лог теперь меняют `innerHTML` только при реальном изменении markup, поэтому обновление метрик больше не должно вызывать видимое моргание всего интерфейса.
- Active-state узла сделан заметнее, но мягче: выбранная строка получила более точный тематический border/background и спокойный акцентный rail вместо излишне резкого выделения.
- Для встроенного desktop-окна расширена базовая и средняя ширина колонки подписок, чтобы URL-поле, подписи и карточки подписок лучше использовали доступное горизонтальное пространство.
- В следующем проходе колонка подписок ещё немного расширена, а действия внутри карточки подписки подняты выше, чтобы карточка выглядела компактнее и визуально плотнее.
- Список узлов перестроен в плиточную сетку с несколькими карточками в ряд, чтобы правая панель лучше использовала ширину desktop-webview.
- Служебный текст по работе с узлами убран из постоянной выдачи под кнопку `?`, чтобы заголовок панели не шумел без запроса пользователя.
- Метрики скорости и трафика переведены на короткие русские подписи `вход` / `выход`, а сами metric-chip стали устойчивее к переполнению по ширине.
- Metric-chip затем дополнительно расширены и перестроены в split-layout с явным вертикальным разделителем и цветовым акцентом для `вход` / `выход`.
- Кнопка `Ping` перенесена из нижней части карточки узла в верхний ряд рядом со статусными бейджами.
- Верхние context-pill в шапке переложены из одной строки в вертикальный стек, чтобы служебные статусы не расползались по ширине.
- Подсказка лог-панели убрана из постоянной выдачи под кнопку `?`, по аналогии с панелью узлов.
- Добавлены проверки `tests.test_embedded_webview` и дополнен `tests.test_gui_server`.
- Пользователь провёл живой smoke встроенного окна в своей графической сессии и подтвердил итоговый desktop UI после нескольких проходов полировки.
- Остаточный риск сужен до fallback-пути на внешний браузер для окружений без `GTK/WebKitGTK`, который в этой сессии отдельно не проверялся.
