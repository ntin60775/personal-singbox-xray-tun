## Исторический контекст

### Изменения

- Перестроить UI вокруг пользовательских задач, а не файлов:
  - `Подписки`;
  - `Узлы`;
  - `Активное подключение`;
  - `Диагностика`;
  - `Настройки`.
- Добавить ясные статусы:
  - подписка обновлена / не обновлена;
  - активный узел;
  - стек запущен / остановлен;
  - последняя ошибка запуска или обновления.
- Убрать из primary UX акцент на ручной правке `xray-tun-subvost.json`.
- Ввести строковый слой локализации:
  - исходно русский;
  - подготовка к английскому без смешивания текстов по коду.
- Подготовить onboarding-first сценарии:
  - добавить ссылку;
  - добавить подписку;
  - выбрать узел;
  - нажать `Старт`.
- Определить, какие инженерные детали остаются видимыми только в `Диагностике`, а не в основном экране.

## Исторические проверки

- Ручной smoke по основному happy path без обращения к shell и JSON.
- Проверка, что ошибки парсинга и сетевые ошибки видны человеку без чтения логов.
- Проверка, что локализационный слой не ломает текущую совместимость с backend API и не размазывает тексты по нескольким файлам без структуры.

## Исторические допущения

- Сначала UX и локализация должны опираться на существующий backend-контур, а не требовать полной переписки runtime-логики.
- Английский нужен как второй язык, но основной язык продукта и документации для этого репозитория остаётся русский.
- Потребительский UX не должен скрывать важные runtime-ошибки DNS/TUN; он должен делать их понятнее и безопаснее, а не менее заметными.
