## Исторические допущения

- Встроенный config означает локальный operator-managed файл `xray-tun-subvost.json`, который может содержать live-значения вне Git.
- Даже при выбранном `builtin` store с профилями и подписками продолжает существовать: пользователь может готовить/обновлять узлы заранее, не делая их текущим runtime.
- Для этой итерации не требуется выносить HTML/CSS/JS из `gui_server.py` в отдельные ассеты, если текущую связность можно сохранить без ухудшения поддержки.

## Исторический итог

- В backend/store добавлен явный `runtime_preference` с режимами `store` и `builtin`, а shell-слой теперь понимает этот выбор и отражает его в runtime-state без автозапуска стека.
- Web UI перестроен вокруг более ясного сценария: появился заметный переключатель источника runtime, локальный feedback по подпискам и более явные тексты о том, что используется сейчас и что пойдёт в следующий старт.
- На реальной ссылке `https://example.com/subscription` подтвержден безопасный fetch и парсинг без конфликта с текущим продовым VPN: upstream отдал `2` валидные строки в `base64`, после dedup сохранился `1` уникальный узел и это теперь явно показывается в ответе backend/UI вместе с числом схлопнутых дублей.
- Подтверждены безопасные проверки без запуска локального `Старт/Стоп`: unit-тесты, shell syntax-check, `py_compile`, HTTP smoke локального GUI backend, добавление подписки и ручное переключение `builtin/store`.
