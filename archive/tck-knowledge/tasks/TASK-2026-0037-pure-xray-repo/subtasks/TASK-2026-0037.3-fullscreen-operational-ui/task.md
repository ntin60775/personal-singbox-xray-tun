## Текущий этап

Подзадача завершена: backend расширен узким operational API для ping и live-метрик, а главный экран пересобран в fullscreen layout `topbar + subscriptions + nodes + log` без вертикальной прокрутки страницы. Дополнительно shell получил фиксированную базовую ширину и центрирование, чтобы интерфейс не растягивался на всю ширину широких экранов. Автоматические проверки прошли; отдельно остаётся только ручная верификация на целевой машине с живым `tun0` и реальными узлами.

## Критерии готовности

- основной экран влезает в полную высоту без вертикального page-scroll;
- на экране остались только runtime-действия, подписки, узлы, лог и базовые индикаторы;
- выбор узла активирует его сразу;
- есть явное действие ping по узлу;
- ошибки явно пишутся в лог и не прячутся в вторичную панель;
- тесты и task-артефакты синхронизированы.

## Итог

В `gui/gui_server.py` добавлены базовые live-метрики по интерфейсу `tun`: текущие счётчики `RX/TX`, расчёт скоростей по дельте между polling-снимками и in-memory cache результатов ping по узлам. Отдельный endpoint `/api/nodes/ping` выполняет реальный TCP-connect до адреса и порта узла и возвращает измеренную задержку в миллисекундах.

В `gui/main_gui.html` экран пересобран в жёсткий fullscreen operational UI без вертикального page-scroll: сверху остались только статус, активная подписка, активный узел, индикатор подписок, скорость, трафик и кнопки `Старт` / `Стоп` / `Диагностика`; в рабочей зоне — только список подписок по URL и список узлов выбранной подписки; внизу — постоянный log feed с фильтром ошибок и явным error banner. На широких экранах весь shell теперь центрируется и ограничивается фиксированной базовой шириной, поэтому интерфейс сохраняет более плотную и собранную композицию.

Выбор строки узла теперь сразу активирует его, а отдельная кнопка `Ping` обновляет кэш задержек без переходов на другую страницу. Из главного экрана убраны маршрутные, readiness- и runtime-карточки, ручной импорт и вторичные диагностические панели. Локально пройдены `python3 -m unittest tests.test_gui_server`, `python3 -m py_compile gui/gui_server.py gui/subvost_runtime.py gui/subvost_store.py gui/subvost_parser.py`, а также HTTP smoke на временном backend `127.0.0.1:8422`. Полноценный browser smoke через Playwright в этой итерации упёрся в локальную недоступность `playwright-cli`, поэтому живую визуальную верификацию нужно отдельно подтвердить на машине пользователя.
