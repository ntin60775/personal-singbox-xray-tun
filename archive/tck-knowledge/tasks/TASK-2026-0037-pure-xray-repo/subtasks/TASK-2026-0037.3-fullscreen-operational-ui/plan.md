## Цель

Свести web UI к fullscreen operational surface без вертикальной прокрутки страницы: верхняя строка состояния, список подписок, список узлов, runtime-действия и постоянный лог с ошибками.

## Границы

### Входит

- новый fullscreen layout для `gui/main_gui.html`;
- ping узла как отдельное UI-действие;
- live-метрики трафика/скорости в status payload;
- явный log feed и отображение ошибок;
- обновление тестов и knowledge-артефактов.

### Не входит

- новые бизнес-сущности в store;
- новая навигация по страницам;
- тяжёлый мониторинг сети вне текущего backend-процесса.

## Планируемые изменения

### Код

- расширить `collect_status()` базовыми сетевыми счётчиками и скоростями по `tun0`;
- добавить endpoint для ping активируемого/выбранного узла;
- пересобрать `gui/main_gui.html` в layout `topbar + subscriptions + nodes + log`;
- ограничить ширину fullscreen-shell и центрировать его на широких экранах;
- убрать вторичные описательные блоки и крупные diagnostic cards;
- перевести выбор узла на немедленную активацию по клику строки;
- писать результаты действий и ошибки в единый видимый лог.

### Конфигурация / схема данных / именуемые сущности

- store оставить прежним;
- status payload расширить полями `traffic` и `ping`.

### Документация

- зафиксировать новую подзадачу в `knowledge/tasks/registry.md`;
- после завершения синхронизировать итог с родительской задачей.

## Риски и зависимости

- fullscreen layout легко сломать переполнением длинных имён и логов;
- ping и traffic нельзя делать декоративными, данные должны быть реальными;
- важно не потерять текущие рабочие сценарии подписок и активации узлов.
