## Границы

### Входит

- полировка toolbar/action-row на основном экране `gui/main_gui.html`;
- отображение времени активного подключения в верхней строке состояния;
- локальный dismiss для баннера последней ошибки;
- минимальное расширение runtime-state/backend payload под таймер подключения;
- обновление task-артефактов и реестра.

### Не входит

- новый layout экрана или новая навигация;
- очистка самого action-log от ошибок на backend;
- полный ручной smoke runtime с реальным `tun0`, если он требует `sudo`.

## Текущий этап

Подзадача завершена. В `run-xray-tun-subvost.sh` добавлена метка `STARTED_AT` для успешного runtime-сеанса, backend пробрасывает её в status payload, а `gui/main_gui.html` использует это значение для компактного таймера подключения в верхней панели. Дополнительно action-кнопки выровнены по ширине и высоте, а баннер последней ошибки получил явный dismiss с повторным показом только при новой ошибке.

- при активном runtime пользователь видит длительность текущего подключения;
- баннер последней ошибки можно скрыть без очистки лога;
- доступные локальные проверки пройдены, task-артефакты синхронизированы.

## Итог

Toolbar основного fullscreen UI доведён под свежий скриншотный фидбек: action-группа теперь состоит из трёх одинаковых по ширине кнопок с увеличенной высотой, а слева от неё появился отдельный чип времени подключения. Значение строится от реального `STARTED_AT`, который shell сохраняет только после успешного старта runtime, поэтому UI не показывает выдуманный таймер в остановленном состоянии.

Баннер `Последняя ошибка` теперь можно скрыть локально кнопкой `×`, не затрагивая сам лог. Если backend отдаёт новую ошибку или следующий запрос `/api/store` завершается новым сбоем, баннер появляется снова. Локально пройдены `bash -n *.sh`, `bash -n libexec/*.sh`, `bash -n lib/*.sh`, `python3 -m py_compile gui/gui_server.py gui/subvost_runtime.py gui/subvost_store.py gui/subvost_parser.py` и `python3 -m unittest tests.test_gui_server`.

Остаточный риск: живой ручной smoke с реальным `sudo ./run-xray-tun-subvost.sh`, обновлением таймера после `Старт` и повторным появлением ошибки на целевой машине в этой сессии не выполнялся.
