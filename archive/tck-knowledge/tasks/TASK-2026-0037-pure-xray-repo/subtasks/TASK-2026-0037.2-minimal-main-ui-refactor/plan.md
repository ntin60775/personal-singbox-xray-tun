## Цель

Свести основной web UI к минимальному рабочему экрану: убрать sidebar-layout, уменьшить повтор статусов, сделать mobile-first порядок блоков и сохранить полный набор текущих действий.

## Границы

### Входит

- новый shell для main GUI;
- упрощение layout и текста секций;
- адаптация inline JS к новой DOM-структуре;
- обновление тестов и knowledge-артефактов.

### Не входит

- новые backend endpoints;
- функциональные изменения parser/store/runtime;
- смена файла-asset без необходимости.

## Планируемые изменения

### Код

- пересобрать `gui/main_gui.html` в компактный layout без постоянного sidebar;
- оставить основные блоки: текущее состояние, маршрут, источники, диагностика;
- убрать JS, связанный с sidebar-навигацией и hash-driven active-state;
- оставить только один GUI entrypoint в backend без дополнительных UI-маршрутов;
- сохранить существующие ID и действия там, где это снижает риск регрессий.

### Конфигурация / схема данных / именуемые сущности

- новых runtime-сущностей не добавлять;
- текущий единый HTML asset оставить корнем web-интерфейса.

### Документация

- добавить подзадачу в `subtasks/`;
- обновить `knowledge/tasks/registry.md`;
- после завершения синхронизировать итог с родительской задачей.

## Риски и зависимости

- крупная правка HTML/CSS/JS может оборвать привязки к существующим DOM-элементам;
- важно не потерять рабочие сценарии импорта, подписок и диагностики в погоне за визуальной чистотой;
- markdown-артефакты нужно прогнать через localization guard перед завершением.
