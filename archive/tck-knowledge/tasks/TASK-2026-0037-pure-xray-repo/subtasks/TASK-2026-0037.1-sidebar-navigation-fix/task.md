## Границы

### Входит

- удаление верхнего бренд-блока и нижнего status-блока из sidebar;
- превращение пунктов меню в реальные интерактивные элементы;
- привязка пунктов меню к рабочим секциям текущего single-screen интерфейса;
- регрессионная проверка HTML-ассета на наличие рабочей боковой навигации.

### Не входит

- новый редизайн всей web-GUI;
- добавление второй HTML-страницы или отдельного роутера;
- переписывание backend API под tabbed-navigation.

## Итог

Из sidebar убраны верхний бренд-блок и нижний status-блок, из-за которых колонка выглядела как статический макет и занимала лишнее место. Вместо них осталась только рабочая навигация по ключевым секциям интерфейса.

Пункты меню переведены на реальные `button`-элементы с привязкой к нужным участкам single-screen интерфейса: обзор, профили, узлы, соединения, проверка, диагностика и блок подписок. При клике меню теперь меняет active-state, прокручивает экран к целевому участку и кратко подсвечивает соответствующую рабочую зону.

Для защиты от регрессии обновлён `tests/test_gui_server.py`: тест теперь проверяет наличие новой sidebar-навигации, отсутствие удалённых legacy-блоков и наличие JS-связывания для перехода по секциям. Дополнительно локально пройдены `python3 -m unittest tests.test_gui_server`, `python3 -m py_compile ...`, `bash -n` для shell-скриптов и `node --check` для inline-скрипта из `design_review.html`.
