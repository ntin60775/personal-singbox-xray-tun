## Цель

Сделать боковую колонку текущей основной web-GUI компактной и рабочей: убрать лишние блоки и дать пользователю реальные кликабельные переходы по экрану.

## Границы

### Входит

- чистка layout sidebar;
- замена статических `div`-пунктов меню на интерактивные элементы;
- JS-привязка меню к нужным секциям интерфейса;
- обновление knowledge-артефактов и регрессионных тестов.

### Не входит

- новая IA всего интерфейса;
- backend-перестройка API;
- отдельный routing layer поверх текущего HTML-ассета.

## Планируемые изменения

### Код

- убрать лишние sidebar-блоки из `gui/design_review.html`;
- добавить интерактивную навигацию по рабочим секциям экрана;
- синхронизировать JS с новой структурой sidebar;
- добавить regression assertions в `tests/test_gui_server.py`.

### Конфигурация / схема данных / именуемые сущности

- новых конфигов и runtime-переключателей не добавлять.

### Документация

- создать подзадачу в `subtasks/`;
- добавить строку в `knowledge/tasks/registry.md`;
- при необходимости коротко обновить родительскую задачу.

## Риски и зависимости

- нужно не оставить JS-ссылок на удалённые DOM-элементы sidebar;
- навигация должна быть полезной в текущем single-screen layout, а не просто менять декоративный active-state;
- markdown-артефакты нужно прогнать через localization guard перед завершением.
