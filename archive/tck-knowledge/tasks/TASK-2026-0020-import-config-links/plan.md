## Исторический контекст

### Изменения

- Ввести локальную нормализованную модель импортированного узла:
  - исходная ссылка;
  - тип протокола;
  - нормализованные transport/security-поля;
  - человекочитаемое имя;
  - признак валидности и причина отказа.
- Реализовать parser-модуль в Python рядом с текущим backend, без зависимости от web UI как единственного потребителя.
- Поддержать каналы импорта:
  - вставка одной ссылки в GUI;
  - вставка нескольких ссылок списком;
  - импорт из локального текстового файла как secondary flow.
- Явно разделить:
  - поддерживаемые схемы, которые конвертируются в runtime-модель bundle;
  - неподдерживаемые схемы и поля, которые должны завершаться понятной ошибкой без молчаливого частичного импорта.
- Определить, где хранить импортированные данные:
  - не в repo-managed `xray-tun-subvost.json`;
  - в operator-managed локальном store вне Git, чтобы не коммитить ссылки с приватными параметрами.
- Добавить backend API и GUI-действия:
  - `Импортировать ссылку`;
  - `Проверить и сохранить`;
  - `Сделать активным`.

## Исторические проверки

- Локальные smoke-кейсы на санитизированных примерах `vless://`, `vmess://`, `trojan://`, `ss://`.
- Негативные кейсы: битая base64-часть, неизвестный transport, отсутствующее обязательное поле, неподдерживаемый протокол.
- Ручной smoke: импортировать ссылку через GUI, выбрать её активной и убедиться, что runtime-конфиг генерируется без ручного редактирования JSON.

## Исторические допущения

- В первой итерации поддерживается только тот поднабор полей, который можно надёжно отобразить в текущую runtime-модель `xray + sing-box` bundle.
- Экзотические схемы, multi-hop, chained proxies и platform-specific флаги могут быть отложены до отдельной задачи.
- Этот черновик считается самым приоритетным из UX-заимствований, потому что он напрямую убирает ручной труд при обновлении узлов.
