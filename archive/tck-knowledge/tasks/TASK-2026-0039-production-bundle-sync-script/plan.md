## Цель

Сделать поддерживаемый и повторяемый сценарий развертывания bundle в внешний каталог без привязки к текущему git-репозиторию.

## Границы

### Входит

- публичный wrapper `install-or-update-bundle-dir.sh`;
- внутренняя реализация sync/deploy в `libexec/`;
- безопасная синхронизация текущего дерева в production-каталог пользователя;
- обновление README и knowledge-артефактов.

### Не входит

- новые системные зависимости ради deploy-сценария;
- destructive mirror-режим с автоматическим удалением stale-файлов;
- ручное тестирование сетевого runtime после deploy.

## Планируемые изменения

### Код

- добавить wrapper и `libexec`-скрипт для install/update bundle в произвольный абсолютный каталог;
- сделать sync неразрушающим: переносить bundle-файлы, но не чистить `logs/` и repo-local мусор;
- после sync актуализировать desktop launcher внутри целевого каталога.

### Конфигурация / схема данных / именуемые сущности

- новый deploy-сценарий принимает target через аргумент или env и требует абсолютный путь.

### Документация

- описать новый deploy-сценарий в README;
- добавить задачу в `knowledge/tasks/registry.md`;
- после завершения зафиксировать факт production-синхронизации.

## Риски и зависимости

- production-каталог находится вне writable-root и потребует отдельного разрешения на запись;
- скрипт не должен случайно копировать `.git`, `.codex`, локальные логи и прочий repo-local мусор;
- при переносе bundle в другой путь нужно сохранить корректный `Icon=` в desktop launcher.

## Проверки

### Что можно проверить кодом или тестами

- `bash -n *.sh`
- `bash -n libexec/*.sh`
- `bash -n lib/*.sh`
- `python3 -m py_compile gui/gui_server.py gui/subvost_runtime.py gui/subvost_store.py gui/subvost_parser.py`
- `python3 -m unittest tests.test_gui_server`

### Что остаётся на ручную проверку

- фактический sync в production-путь;
- ручной запуск `open-subvost-gui.sh` и runtime smoke из production-каталога.
