# Проверка локализации Markdown

## Когда использовать

Использовать, когда задача затрагивает:
- новые или измененные `.md`-документы;
- roadmap, архитектурные заметки, отчеты и планы;
- ревью документации на предмет русскоязычной локализации.

## Цель

Перед завершением изменений убедиться, что Markdown-документы:
- содержат русскоязычную прозу;
- не состоят в основном из англоязычных предложений;
- сохраняют английский только для машинно-значимых литералов, команд, путей, URL, ID и технических терминов.

## Базовый порядок

1. Определить scope проверки:
   - если менялись конкретные `.md`-файлы, проверять их адресно;
   - если меняется много документации, проверять весь релевантный корень проекта.
2. Запустить глобальный CLI:

```bash
check-docs-localization
check-docs-localization README.md docs/roadmap/master-roadmap.md
```

3. Если есть нарушения:
   - исправить англоязычную прозу;
   - оставить английский только там, где он действительно машинно-значим;
   - повторно прогнать проверку.

## Правила интерпретации

- Команды, пути, URL, идентификаторы и названия внешних проектов не считаются нарушением сами по себе.
- Если документ почти полностью англоязычный и не содержит русскоязычной прозы, это считается ошибкой.
- Если в строке есть длинный англоязычный фрагмент без кириллицы, это подозрительная строка и ее нужно проверить вручную.

## Output

В финальном отчете коротко указать:
- что именно проверялось;
- прошла ли локализация;
- если были исправления, какие документы пришлось править.
