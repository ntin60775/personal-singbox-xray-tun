# Эталонный формат файла с трудозатратами

Используй этот файл только как ориентир по компактному формату. Файл с часами должен быть короче основного отчета и не повторять его текст.

## Пример
```markdown
# Оценка трудозатрат (часы) — bus-order-status-sync

## Итого
10 ч

## Основные статьи
- Разбор текущего сценария обмена и связанных ограничений: 2 ч
- Изменение серверной логики загрузки и проверок: 4 ч
- Проверка сценария загрузки и регрессионные ручные проверки: 2 ч
- Подготовка отчета по доработке и согласование: 2 ч

## Допущения
- Изменения не требуют отдельной миграции данных.
- Соседние сценарии обмена используют тот же набор базовых объектов и не требуют отдельной ветки реализации.

## Что может сдвинуть оценку
- Появление дополнительных веток обмена с отдельными правилами статусов.
- Необходимость разбирать уже созданные документы после внедрения.
```
